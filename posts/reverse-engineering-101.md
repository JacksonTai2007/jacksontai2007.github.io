# 逆向入门：用 IDA Pro 啃下第一个 CrackMe

CrackMe 是逆向练习里最经典的入门题型：一个小程序，让你输入「密码」或「序列号」，猜对了弹出 *Congratulations*，猜错了骂你一句。目标就是：**不靠爆破，靠分析，把正确答案找出来。**

这篇记录我做第一个 CrackMe 的完整思路，顺便把新手最容易踩的几个坑列出来。

## 先跑一遍，建立直觉

逆向的第一步永远不是打开 IDA，而是**先把程序当普通用户跑一遍**：

- 它要我输入什么？一个字符串？一串数字？
- 输错了提示是什么？「Wrong serial」之类的字样，往往就是我们在二进制里搜索的锚点。

记住那句错误提示，这是最省事的突破口。

## 静态分析：定位关键逻辑

把程序拖进 IDA Pro，等它分析完，按下面的顺序走：

1. **Shift+F12** 打开 Strings 窗口，搜刚才记下的提示文字，比如 `Wrong`；
2. 双击字符串，跳到它在数据段的位置；
3. 在它上面按 **X**（查看交叉引用），看谁用到了这个字符串 —— 那段代码基本就是校验逻辑所在。

跳过去之后按 **F5** 反编译成伪 C 代码，可读性会比纯汇编高一个数量级：

```c
int check(char *input) {
    int len = strlen(input);
    if (len != 8) return 0;          // 长度必须是 8
    for (int i = 0; i < len; i++) {
        if ((input[i] ^ 0x20) != key[i])  // 逐字节异或比对
            return 0;
    }
    return 1;                         // 全部通过
}
```

读到这段，逻辑就清楚了：输入长度得是 8，每个字符异或 `0x20` 之后要等于内置的 `key` 数组。

## 动态调试：验证猜想

光看代码容易看走眼，**动态调试**能让你确认。在 `check` 函数入口下断点（**F2**），运行（**F9**），随便输个 `12345678`：

- 断下来后单步（**F8**）走到比较那一行；
- 在寄存器 / 内存窗口里看 `key[i]` 实际的值；
- 把每个字节异或回去（`key[i] ^ 0x20`），拼起来就是答案。

> **小技巧**：IDA 的 *Appcall* 或者直接在调试器里改 `eax`/`zf` 标志位，可以强行让校验通过 —— 但那叫 patch，不叫解题。CrackMe 的乐趣在于**算出**答案。

## 新手最容易踩的坑

我自己栽过的几个：

1. **没区分有符号 / 无符号比较**。`jg` 和 `ja` 看着差不多，符号搞反了整个逻辑就反了。
2. **忽略了字符串结尾的 `\0`**。`strlen` 是 8，但缓冲区比较时可能算上了终止符。
3. **被花指令 / 反调试劝退**。入门题一般没有，但一旦遇到 `IsDebuggerPresent` 之类，先别慌，把它 NOP 掉或者改返回值即可。
4. **过度依赖爆破**。8 位可见字符的空间是 95⁸，约等于 6×10¹⁵，硬爆是不现实的 —— 这也是逆向存在的意义。

## 小结

一个标准的 CrackMe 求解流程：

| 阶段 | 工具 / 操作 | 目的 |
| --- | --- | --- |
| 试运行 | 直接跑 | 找错误提示当锚点 |
| 静态分析 | Strings → Xref → F5 | 定位并读懂校验逻辑 |
| 动态调试 | 断点 → 单步 → 看内存 | 验证猜想、取出 key |
| 求解 | 反推算法 | 算出正确输入 |

逆向没有魔法，全是「观察 → 假设 → 验证」的循环。第一个 CrackMe 做完，你就摸到门了。下次写写 Android 上的 so 层逆向 👋
