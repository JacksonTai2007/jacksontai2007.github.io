# 把 GPT Image 2 提示词工程做成一个 Claude Skill

折腾 AI 画图久了，我攒下一个越来越强的执念：**同样的画面想法，为什么每次都要重新拼提示词？** 于是我把自己的一套方法整理开源了 —— [gpt-image-2-prompting-skill](https://github.com/JacksonTai2007/gpt-image-2-prompting-skill)，一个把 GPT Image 2（OpenAI 的文生图 / 图生图模型）提示词工程系统化的 **Claude Skill**。

这篇聊聊：它解决什么问题、内部怎么组织、以及你怎么上手。

## 为什么做这个

网上的「awesome prompt」合集大多是一堆一次性示例。只要你的需求稍微变一点，就又回到了对着输入框瞎猜的状态。

这个 skill 反着来：它不堆示例，而是把这些示例背后的**模式**压缩成「一套结构化协议 + 一批即用配方」，让 Claude 能针对**你的具体任务**写出强提示词，并且告诉你每一部分为什么这么写。

它想扭转的核心观念是一句话：

> **别再写散文式的许愿，开始写结构化的规格说明。**

GPT Image 2 能跟随又长又分层的指令，所以瓶颈早就不是「它能不能画出来」，而是「**你能不能把画面描述得足够精确，精确到可复现**」。

## 里面有什么

仓库结构很清楚 —— skill 本体就是一份 `SKILL.md` 加一个 `references/` 资源目录：

```text
gpt-image-2-prompting/
├── SKILL.md                  核心方法：发力点、六段式协议、通用原则、
│                             国际化、图生图、系列/批量编排、运行时旋钮
└── references/
    ├── categories.md         请求路由器 → 产物原型 → 模板 + 配方
    ├── templates.md          13 类产物 —— 散文 + JSON 模板 + 避坑指南
    ├── scene-cookbook.md     29 个调好的即用场景配方，每个都附「关键技巧」
    ├── style-library.md      可组合菜单：画幅、镜头、胶片、灯光、调色板、
    │                         材质、艺术风格、构图、写实增强
    └── troubleshooting.md    失败生成的「症状 → 成因 → 修复」对照表
```

工作流是闭环的：当你要一条图片提示词时，skill 会先**给请求分类**（靠 `categories.md`），取出匹配的模板 + 配方，填进下面这六块；如果出图不对，再对照排错表去 debug。

### 六段式提示词协议

每一条强提示词 —— 不管是散文式还是 JSON —— 都由这六块构成，顺序是「意图 → 结构 → 风格 → 约束」：

1. **Intent（意图）** —— 这是什么产物、拿来干嘛
2. **Subject（主体）** —— 画面主角，拆解成一个个具体属性
3. **Composition（构图）** —— 画幅比例、布局/网格、模块数量、相机 + 镜头、焦点层次
4. **Text（文字）** —— **确切**的字符串，每条都带角色 + 拼写/可读性/语言锁定
5. **Style（风格）** —— 媒介、笔触/渲染、材质、色彩体系、灯光
6. **Constraints（约束）** —— 明确的负面项与禁止出现的元素

顺序本身就是设计的一部分：先定「要什么」，再定「长什么样」，最后才划「不要什么」。

### GPT Image 2 特别擅长什么（以及 skill 怎么利用）

整套方法是围绕这个模型的真实强项设计的，每一条都配了「怎么用」：

| 强项 | 怎么利用 |
| --- | --- |
| **文字排版** —— 能渲染确切文字（拉丁/西里尔/中日韩） | 把字面字符串硬编码进去 |
| **身份保持** —— 同一角色跨多个场景 | 开头就声明「契约」；要严格锁定就配参考图 |
| **结构化设计** —— 信息图、爆炸图、网格 | 指定确切数量和骨架 |
| **风格归纳** —— 抓住一种美学的「魂」 | 点名风格，或描述它的运作机制 |
| **世界知识** —— 真实地点、知名游戏/应用/品牌 | 直接点名来复刻那种观感 |
| **提示词遵循** —— 详尽提示词反而比简短的效果好 | 写得具体、写得「苛刻」 |
| **照片级写实** —— 跟照片/截图难辨真假 | 用相机/胶片术语 + 加点瑕疵感 |

最后这条很反直觉：很多模型你写太多它会乱，而 GPT Image 2 是**喂得越细、回报越高**。六段式协议正是为了让你「有条理地写很多」。

## 上手

### 安装

**Claude Code / 会读 skills 目录的 agent：**

```bash
git clone https://github.com/JacksonTai2007/gpt-image-2-prompting-skill.git
cp -r gpt-image-2-prompting-skill/gpt-image-2-prompting ~/.claude/skills/
# 重启 Claude Code 会话
```

**Claude.ai / 桌面端：** 下载 `dist/gpt-image-2-prompting.skill`，到 **设置 → Capabilities/Skills** 里上传。

> 不同套餐和 App 版本里 skill 的入口和菜单名可能不一样，找不到就查一下 Anthropic 最新文档。

### 怎么用

装好之后不用记任何命令，正常说人话就行，skill 会自动激活。比如：

```text
帮我写一条 GPT Image 2 提示词：极简风智能手表发布海报，标语 "Time, refined"
做个 Instagram 美妆人像系列的提示词 —— 同一个人，跨 6 个场景
我的 AI 图按钮上老是出乱码文字，帮我修提示词
把这张角色草图变成官方三视图设定稿（图生图）
```

你会拿到一条结构化的提示词（散文或 JSON），外加每一块背后的理由 —— 这样你自己想改也知道动哪里。

### 两个设计细节

- **国际化**：skill 从不硬编码某种人群默认值。当画面里有人、但外貌没指定时，它会跟随场景的文化语境，否则匹配用户的语言/地区，并且始终把外貌做成「一处可改」的变量。
- **系列生成**：要一组图时，它会产出一条批量编排的提示词（「N 张独立的图，别合并成网格」）；同时它很诚实地告诉你 —— 跨多次独立生成要严格保持同一身份，仍然需要参考图。
