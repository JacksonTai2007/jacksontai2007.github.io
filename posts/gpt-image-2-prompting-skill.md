# 把 GPT Image 2 提示词工程做成一个 Claude Skill

折腾 AI 画图久了，我攒下一个越来越强的执念：**同样的画面想法，为什么每次都要重新拼提示词？** 于是这个周末我把自己的一套方法整理开源了 —— [gpt-image-2-prompting-skill](https://github.com/JacksonTai2007/gpt-image-2-prompting-skill)，一个把 GPT Image 2（也就是 DALL·E 3）提示词工程系统化的 **Claude Skill**。

这篇就聊聊：它解决什么问题、我是怎么设计的、以及你怎么上手。

## 先说痛点

网上的提示词资源大多是「示例的堆叠」——某某大神放出 50 条惊艳的 prompt，你照抄能出图，但只要需求稍微一变（换个画幅、加段文字、改个风格），你就又回到了对着输入框瞎试的状态。

问题不在示例不够多，而在**没有结构**。示例是「结果」，你真正需要的是能稳定复现这些结果的「方法」。

所以这个 skill 的核心不是又一份提示词大全，而是把社区里逆向出来的有效模式，压缩成一句话：

> **一套结构化协议 + 一批即用配方。**

## 这是什么

它是一个 Claude Skill —— 装好之后，当你向 Claude 提出「帮我写个 XX 的 AI 画图提示词」这类请求时，它会自动激活，用一套固定方法帮你把模糊想法翻译成精确、可复现的提示词。

不是插件、不需要写代码，本质是一份结构化的方法论 + 资源库，喂给 Claude 当「专业技能」。

## 核心：六段式协议

整个方法的地基，是一条**有顺序**的提示词结构。每一条强提示词都按这六块来写：

1. **Intent（意图）** —— 产物是什么、用来干嘛（海报？图标？信息图？）
2. **Subject（主体）** —— 画面主角，带具体属性（不是「一只猫」，而是「一只蜷缩的橘猫，琥珀色眼睛」）
3. **Composition（构图）** —— 画幅比例、布局、相机/镜头、视觉焦点层次
4. **Text（文字）** —— 要出现的确切字符串，并锁定拼写与语言
5. **Style（风格）** —— 媒介、材质、色彩、灯光
6. **Constraints（约束）** —— 明确的负面项，禁止出现什么

顺序本身就是设计的一部分：先定「要什么」，再定「长什么样」，最后才划「不要什么」。这个次序能让模型先抓住主干，再填细节，避免风格词淹没主体。

## 配套资源

光有协议还不够，所以仓库里带了一整套即用资源：

| 文件 | 内容 |
| --- | --- |
| `SKILL.md` | 核心方法、通用原则、国际化、图生图工作流 |
| `templates.md` | 13 类产物模板，每类都有 prose（散文式）和 JSON 两种写法 |
| `scene-cookbook.md` | 29 个调好的即用场景配方 |
| `style-library.md` | 可组合的菜单：画幅、镜头、胶片、灯光、调色板、风格 |
| `troubleshooting.md` | 「症状 → 修复」诊断表 |

`style-library` 我个人最爱 —— 它把风格拆成一个个可拼装的零件。想要「35mm 镜头 + 柯达胶片 + 黄昏侧光 + 莫兰迪色」？从菜单里各挑一个拼起来就行，不用每次回忆那些咒语般的关键词。

## 它吃透了 GPT Image 2 的哪些强项

方法是围绕这个模型的真实能力设计的，重点压在它擅长的地方：

- **精确文字排版** —— 能在图里渲染指定文字，且支持拉丁/西里尔/中日韩
- **身份保持** —— 同一个人物/角色能跨多个场景保持一致
- **结构化设计** —— 信息图、网格、爆炸分解图这类有章法的版面
- **风格归纳** —— 抓住一种美学的「魂」，而不只是表面滤镜
- **真实世界知识** —— 认识真实的地点和品牌
- **越详尽越好** —— 对又长又挑剔的提示词，它反而表现更好
- **照片级写实**

最后一条很关键，也颠覆直觉：很多模型你写太多它会乱，而 GPT Image 2 是**喂得越细、回报越高**。六段式协议正是为了让你「有条理地写很多」。

## 上手教程

### 安装

**Claude Code / agent 用法：**

```bash
git clone https://github.com/JacksonTai2007/gpt-image-2-prompting-skill.git
cp -r gpt-image-2-prompting-skill/gpt-image-2-prompting ~/.claude/skills/
```

**Claude.ai / 桌面端：** 下载 `dist/gpt-image-2-prompting.skill`，到 设置 → Capabilities/Skills 里上传即可。

### 怎么用

装好之后不用记任何命令，正常说人话就行，skill 会自动激活。比如：

```text
帮我写一条 GPT Image 2 提示词：极简风的智能手表发布海报，主标题是 "Time, refined"
给我做个 Instagram 美妆人像系列的提示词 —— 同一个人，跨 6 个场景
我的 AI 图老是出乱码文字，帮我修一下提示词
把这张角色草图变成官方三视图设定稿
```

### 一个完整示例

感受一下「随手写」和「按协议写」的差距。

随手写：

```text
一张智能手表海报
```

按六段式协议写（skill 帮你扩展成）：

```text
Intent: 一张产品发布主视觉海报，竖版，用于社交媒体。
Subject: 一块极简智能手表，正面视角，金属表壳，深空灰，表盘亮起显示时间。
Composition: 3:4 竖幅，手表居中略偏上，大量负空间，微距镜头质感，浅景深。
Text: 顶部居中大字 "Time, refined"，无衬线字体，确保拼写准确，仅此一行文字。
Style: 高端产品摄影，柔和影棚布光，冷调背景，轻微反光，质感细腻。
Constraints: 不要表带杂乱、不要多余文字、不要花哨背景、不要水印。
```

出图的稳定性和质感完全不是一个量级 —— 而且因为结构固定，下次想改成「横版」或「换标语」，你只动对应那一块就行，其余照旧。这就是「可复现」的意思。

## 设计原则与小结

两条我比较在意的原则：

- **国际化** —— 不硬编码人群默认值。人物外貌应贴合文化语境或用户所在地，而不是默认某一种长相。
- **系列生成** —— 一条提示词就能批量编排出一组图；但要严格保持同一身份，仍建议配参考图（图生图）。
