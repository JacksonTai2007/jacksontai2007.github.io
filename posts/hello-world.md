# Hello World：博客换新装

拖了很久，终于把这个老掉牙的个人主页推倒重做了一遍。新版本是一个**极简黑白风**的技术博客，主打三个字：**简单**。

## 为什么要重做

旧版是套了个现成的单页模板，左边头像、右边几张空卡片，看着挺花哨，但有两个问题：

1. 它**不是博客** —— 没法写文章、没有列表和归档，本质上就是一张名片；
2. 那些卡片我一直懒得填，放着一堆占位的 `i1.png`、`i2.png`，尴尬。

我想要的其实很朴素：一个能安安静静写字、记笔记的地方。于是就有了现在这套。

## 设计取舍

最后定下来的几条原则：

- **极简黑白**：只用黑、白、灰，配一点等宽字体做点缀。耐看，也不会喧宾夺主。
- **深浅色切换**：跟随系统，也能手动切，状态记在 `localStorage` 里。
- **零构建**：没有 Jekyll、没有 Hugo、没有 `npm run build`。纯静态 HTML + 一点点原生 JS，GitHub Pages 直接就能跑。
- **Markdown 写作**：正文用 Markdown，浏览器端用 [marked](https://marked.js.org/) 实时渲染。

> 我也考虑过最流行的 Chirpy 主题，但它要走 GitHub Actions 构建、还得改仓库设置，对一个「就想写点东西」的小站来说太重了。

## 怎么发新文章

整个流程只有两步：

1. 在 `posts/` 目录下新建一个 Markdown 文件，比如 `my-post.md`；
2. 在 `posts/index.json` 的 `posts` 数组里加一条：

```json
{
  "id": "my-post",
  "title": "我的新文章",
  "date": "2026-06-02",
  "category": "随笔",
  "tags": ["标签A", "标签B"],
  "excerpt": "一句话摘要，会显示在列表里。"
}
```

`id` 要和文件名（去掉 `.md`）一致，剩下的字段顾名思义。保存、推送，文章就上线了 —— 首页、列表、归档、标签筛选会自动更新，**不需要改任何代码**。

## 支持哪些写法

常见的 Markdown 语法都能用：

- **加粗**、*斜体*、`行内代码`
- 有序 / 无序列表
- 引用、表格、图片
- 代码块（见上文）
- 二级、三级标题会**自动生成右侧目录**

| 功能 | 状态 |
| --- | --- |
| 深浅色 | ✅ |
| 标签筛选 | ✅ |
| 全文搜索 | ✅（标题/摘要/标签）|
| 目录 TOC | ✅ |
