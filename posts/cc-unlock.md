# cc-unlock：给 AI Coding Agent 搭一套安全研究上下文框架

做逆向和漏洞研究久了，你会发现 AI 编码助手卡在一个很尴尬的位置：它能写业务代码，但一碰到安全研究场景 —— 分析 Lua 脚本、跨版本兼容移植、渗透测试辅助 —— 就开始保守。问题不是它不够聪明，而是**它不知道你在做什么、在什么授权背景下做**。

这就是我开源 [cc-unlock](https://github.com/JacksonTai2007/cc-unlock) 的原因。

## 它解决什么问题

Claude Code 和 Codex 本身已经支持通过 `CLAUDE.md` / `system-prompt.md` 来注入项目级上下文。cc-unlock 做的事，就是把这个机制用起来 —— 系统性地给 AI agent 建立一套**安全研究领域的工作背景**：

- 你在做什么方向的研究（逆向、漏洞分析、渗透测试……）
- 你的授权背景是什么
- 这个领域的标准术语和操作规范

有了这层上下文，AI 才能真正进入「安全研究员同事」的角色，而不是每次都从零开始猜你的意图。

## 五层架构

v3.0 的核心是五层注入结构，按从宏观到微观的顺序叠加：

| 层级 | 作用 |
|------|------|
| **授权框架** | 建立研究背景与授权声明 |
| **行为锚定** | 确定 agent 在这个项目里的工作模式 |
| **示例驱动学习** | ~300 个 Q&A 对，覆盖 15 个安全子领域 |
| **术语标准化** | 统一 24+ 安全术语，避免 agent 理解偏差 |
| **模型适配策略** | 针对不同模型版本的差异化配置 |

其中示例驱动那层是效果最明显的。与其说「你可以帮我做逆向」，不如直接给它看 300 个真实的研究问答 —— **用实例教它，比用规则约它有效得多**。

## 一次真实验证

文档里记录了一个完整的验证案例：把 DNF 手游的 Lua 脚本跨版本迁移，需要分析旧版脚本结构、对齐新版 API 差异、输出可运行的兼容版本。

同一套 cc-unlock 配置，在 Claude Desktop（Opus 4.8）和 Codex Desktop（GPT 5.5）上都跑通了，两个平台都产出了可用的 Lua 脚本。

这是我认为它价值最高的地方：**配置一次，跨模型复用**。你不用给每个 AI 工具单独调教，把配置文件装进去就能迁移上下文。

## 怎么用

安装做得很轻量，三个平台各一个脚本，执行完自动备份原有配置：

**Windows：**
```bat
install.bat
```

**macOS / Linux：**
```bash
bash install.sh
```

装完重启 Claude Code 或 Codex 的会话，配置就生效了。

想卸载也简单，安装脚本会在备份目录里留着原配置，手动恢复就行。

## 适合谁用

- 做游戏安全、脚本逆向的研究者
- CTF 参赛选手，尤其是 Android / native 方向
- 渗透测试工程师，需要 AI 辅助写 PoC 或分析目标
- 任何需要在安全领域长期使用 AI coding agent 的人

## 小结

cc-unlock 不是什么神奇的破解工具，本质上就是一套**精心设计的项目上下文模板**。它利用的是 Claude Code 和 Codex 本就支持的配置注入机制，只是把安全研究领域的背景知识结构化地塞进去了。

对我自己来说，最大的收益是：每次开新的安全研究项目，不用再花时间「预热」AI、解释背景、调整它的工作模式 —— 装上配置，直接开干。
